# DEV-20260226-001 · MVP 基建与登录闭环

## Stage 1 澄清结论
1. 我的理解
- 当前仓库为空，需要先建立可运行前端基建，再打通首条业务闭环。
- MVP 首个业务闭环定义为“登录页 UI + 表单验证 + mock 登录 + 登录后跳转页面”。
2. 边界确认
- 技术栈固定为 React + TypeScript + Tailwind + Zustand + React Router v6。
- 不接真实后端，认证使用 mock service。
- 仅实现登录到 `/map` 的流程，不实现注册/找回密码。
3. 依赖检查
- 无前置代码依赖；需先完成工程初始化与质量门禁。

## Stage 2 设计与任务拆分

### In Scope
- 初始化工程与目录结构。
- 配置 lint/test/build 与 CI。
- 实现登录页、验证逻辑、mock 认证、路由与状态存储。
- 编写核心验证逻辑单元测试。

### Out of Scope
- 真实后端 API 对接。
- 完整业务页面（课程/地图/关卡）实现。
- E2E 测试与发布流水线。

## 风险与应对
1. 风险：Router 与测试环境版本不一致。
- 应对：固定 `react-router-dom` 到 v6，Vitest 使用 jsdom。
2. 风险：后续接真实 API 可能改动登录服务。
- 应对：先定义 `IAuthService` 接口，后续仅替换实现。
3. 风险：样式后续改版导致返工。
- 应对：颜色和交互约束集中在常量与 CSS 变量。
